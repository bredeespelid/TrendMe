Microsoft Fabric April 2025: Oppsummering av Nyheter

Publisert: 1. Mai 2025 | av Brede Espelid | ⏱️ ca. 9 min lesetid

April 2025-oppdateringen for Microsoft Fabric bringer en rekke forbedringer og nye funksjoner. Her er en oppsummering av høydepunktene, inkludert mer tilgjengelig AI, nyheter for Data Warehouse og sanntidsanalyse.

Merk: Microsoft tilbyr også ressurser som gratis sertifiseringstrekninger og læringsøkter for Fabric-brukere.

Generelt

Lavere terskel for AI og Copilot

En viktig nyhet er at kravet for å bruke Copilot og AI-funksjoner i Fabric nå er senket til F2 SKU. Dette gjør avanserte AI-verktøy som Copilot, data agents og AI Functions tilgjengelig for langt flere brukere og team, selv om ytelsen vil være begrenset av kapasitetens størrelse på de minste SKU-ene. Målet er å la flere eksperimentere og øke produktiviteten med AI.

Data Science

Low-code AI i Notebooks (Preview)

En ny AI/ML-fane i Fabric Notebooks gir low-code tilgang til verktøy som Data Wrangler, AutoML og Copilot. Spesielt nyttig er et nytt grensesnitt for å anvende AI-funksjoner (som LLM-basert databerikelse) direkte på pandas eller Spark DataFrames med minimal kodeskriving.

Forbedret Data Wrangler med AI (Preview)

Data Wrangler får snart flere AI-drevne funksjoner:

  • Automatiske forslag: Regelbasert AI foreslår relevante datarensoperasjoner.
  • Copilot for kode: Generer egendefinerte operasjoner ved hjelp av naturlig språk.
  • Pandas til PySpark-oversettelse: AI hjelper med å oversette egendefinert pandas-kode (også Copilot-generert) til PySpark ved eksport.

Data Warehouse

Kolonnemanipulering (GA)

To etterlengtede funksjoner er nå generelt tilgjengelige (GA):

  • ALTER TABLE DROP COLUMN: For enkelt å fjerne kolonner (støttes ikke for Lakehouse-tabeller eller ved tidsreiser før dropping).
  • sp_rename for kolonner: Gir mulighet til å endre kolonnenavn uten nedetid (støttes ikke i Lakehouse).

Sesjonsbaserte #temp-tabeller (GA)

Du kan nå opprette sesjonsomfattende midlertidige tabeller (#temp) for å håndtere mellomliggende resultater i batchjobber. Disse kan være enten:

  • Ikke-distribuerte (mdf-basert): CREATE TABLE #navn (...)
  • Distribuerte (Parquet-basert): CREATE TABLE #navn (...) WITH (DISTRIBUTION=...)

Dette gir fleksibilitet for ulike behov. Se dokumentasjonen for detaljer.

Migration Assistant for Data Warehouse (Preview)

En ny, innebygd assistent forenkler migrering fra Azure Synapse Analytics (Data Warehouse) til Fabric Data Warehouse. Verktøyet hjelper med å vurdere, migrere metadata (med automatisk skjemakonvertering), kopiere data og omdirigere tilkoblinger, støttet av AI-assistanse.

OPENROWSET-funksjonen (GA)

OPENROWSET er nå GA, og lar deg spørre direkte mot Parquet- og CSV-filer i ADLS eller Blob Storage (SELECT * FROM OPENROWSET(BULK 'sti/til/fil.parquet')). Dette er nyttig for å inspisere data før lasting, håndtere partisjonerte data, komplekse Parquet-typer, tilpasse utdataskjema (WITH-klausul) og bruke i CTAS/INSERT SELECT for transformasjon underveis.

BULK INSERT-setningen (GA)

BULK INSERT (BULK INSERT tabell FROM 'sti/til/fil') er nå GA. Den ligner COPY INTO, men dens styrke ligger i kompatibilitet med tradisjonell SQL Server/Azure SQL-syntaks, noe som forenkler migrering av eksisterende T-SQL-baserte lasteskript (inkludert støtte for bcp-formater og kodetabeller).

Real-Time Intelligence (RTI)

OpenAI Plugins for Eventhouse

To nye plugins lar deg bruke OpenAI direkte i KQL:

  • ai_embed_text: Genererer tekst-embeddings.
  • ai_chat_completion: Bruker modeller som ChatGPT for dataanalyse.

Disse er nyttige for RAG-applikasjoner og databerikelse.

Nytt i Kusto Detective Agency

En ny Fabric-spesifikk utfordring ("Digibus") er lansert for å lære RTI-verktøy (Eventstream, Eventhouse, Dashboard, Activator) og KQL gjennom en engasjerende case.

Bedre Azure Monitor-integrasjon i KQL Queryset

Application Insights (AI) og Log Analytics (LA) behandles nå som førsteklasses datakilder i KQL Queryset, på lik linje med Eventhouse og ADX. Dette forenkler spørringer på tvers av disse tjenestene.

Forbedret Low-Code Datautforskning

Flere forbedringer i datautforskningen (i Real-Time Dashboard-fliser og KQL Database):

  • Hierarkisk kolonnepane: Gir oversikt over datatyper og statistikk.
  • Fokusmodus: Fokuser enten på visualisering eller resultatgrid.
  • Høyreklikk-handlinger: Raske handlinger direkte i gridet.
  • Filterbygger med OR: Mer fleksible WHERE-betingelser.
  • Datetime-velger: Enklere valg av dato/tid for filtrering.

Forbedret Overvåking i Eventhouse

Eventhouse System Overview og KQL Database monitoring får flere forbedringer:

  • Ny metrikk for inntatte rader over tid.
  • Ny fane for topp inntatte databaser (viser rader og feil).
  • Forbedret "Topp 10"-visning med cache miss-rate for spørringer.
  • Eventhouse-detaljer er flyttet til et sidepanel fra menylinjen.

Eventstream Real-time Weather Connector

En ny connector for sanntids værdata er tilgjengelig i Eventstream, med en tilhørende demovideo (merk: faktisk video-URL mangler i kildetekst) som viser hvordan den settes opp.

Databases (SQL Database)

Generelle Forbedringer

  • Nye regioner: Støtte for Australia Southeast, Italy North, Japan East, Poland Central, WestUS3.
  • Backup-fakturering: Fra 1. april 2025 faktureres backup-lagring som overstiger tildelt databasestørrelse. Automatiske backuper kjører fortsatt (full ukentlig, diff. hver 12t, logg hvert 10min).
  • Ytelsesdashboard: Viser nå ledende blokkerende spørringer.
  • Terraform, REST API, CLI-støtte: Muliggjør IaC og automatisering for SQL-databaser.
  • Pipeline-integrasjon: SQL DB kan nå brukes som kilde i Stored Procedure/Script-aktiviteter uten manuell tilkoblingsinfo.
  • Grafdatabase-støtte: T-SQL-støtte for å lage og spørre grafdatabaser (noder, kanter) med MATCH-klausulen.

Data Factory

Mirroring for Firewalled Snowflake (Preview)

Det er nå mulig å speile data fra Snowflake-instanser som er beskyttet av en brannmur. Dette krever bruk av en VNet data gateway eller on-premises data gateway for sikker tilkobling.
